Getting Started Guide

A Procedure is a building block you can use to author short-lived, interdependent behaviors in a standardized way. For example, a character combat stance, equipping an item, opening a door for 5 seconds, or entering a tutorial state.

1. Create the Context

Create a new GameObject and add the ProceduresManager component to it.

Tip

The ProceduresManager is what "owns" and runs all the procedures for a specific character or system. Think of it as the car ğŸš— that will hold all the parts.

2. Create the Procedure Asset

  1. In the project window, navigate to the folder you want to store the asset.
  2. Right click and select Create/Procedures/Procedure
  3. Give your Procedure a name, like "Jump".

This asset is like a blueprintğŸ“œ for a part, such as an engine.

3. Connect the Procedure to the Manager

  1. Select the ProceduresManager GameObject in the hierarchy.
  2. In the inspector, find the Source Data list.
  3. Drag your newly created Procedure asset into this list.
Tip

We can only add each Procedure asset once. The ProceduresManager will create a unique instance of this Procedure at runtime. Think of the instance as the engineâš™ï¸ built from the blueprint.

4. Manually Control a Procedure at Runtime

The Weight determines if a procedure is active (starts) or inactive (stops). A weight greater than 0 makes it active, while a weight of 0 makes it inactive.

Test the Procedure's State

  1. In the hierarchy, select the ProceduresManager GameObject.
  2. Enter Play Mode (Ctrl+P).
  3. In the inspector, find your Jump procedure. Notice its status icon is green ğŸŸ¢, meaning it's active by default.
  4. Drag the Weight slider to 0. The status icon should turn grey âšªï¸ (inactive).
  5. Drag the Weight slider back to 1. The status icon will turn green ğŸŸ¢ again (active).
